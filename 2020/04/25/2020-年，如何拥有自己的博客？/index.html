2020 年，如何拥有自己的博客？

发表于 2020-04-25 分类于 技术相关

当下我们表达和写作的权利显然受到严重侵犯，中文互联网世界的人们正失去理智。因此，拥有一个属于自己的、不会被举报和审查的、永久存档的博客便显得十分重要。本文即为自行搭建博客的教程。全程（包括搭建、后续维护和写作）都需要用到电脑。

建议提前阅读 中文互联网中「讨论」的消亡 - 少数派 （本文首发于微信公众号「沙丘研究所」，少数派经授权转载。 阅读原文）并自行学习 MarkDown 语法（我自己都不怎么熟练），后续的写作中均使用 MarkDown，请自行选择 MarkDown 编辑器（其实记事本就是最简单的一个，只不过需要修改文件后缀名），本人使用的是 Typora。

完成后，在 D:/Blog/blog （好吧两个同名目录有点复杂）下，共有以下几个子文件夹：

  • node_modules：npm 依赖包，Hexo 使用该文件夹下的文件渲染网页。

  • _config.yml：博客配置文件。在后文称作“站点配置文件”。

其中，我们需要使用的只有 source，theme，和 _config.yml。

GitHub 用户名 可在 GitHub 首页 - 点击右上角头像 - Signed in as Hexo-Sample 找到。

执行 ssh-keygen -t rsa -C "GitHub 注册邮箱" 后会要求填写密码，建议为空，直接三个回车即可。如果设置密码，可能每次部署都会要求使用密码。

然后，前往 C 盘 - 用户 - 用户名 - .ssh 中找到名为 id_rsa.pub 的文件。用记事本打开该文件，并复制其中的所有内容。前往 SSH and GPG keys ，点击 New SSH key。Title 随意填写，不影响后续操作且在后续命令中也不会用到，Key 一栏填写刚刚复制的内容。在 Git Bash 中输入 ssh git@github.com 检查公钥设置是否成功。

不要急着关闭浏览器，在 GitHub 主界面点击 New repository 创建新仓库，仓库名填写 Hexo-Sample.github.io。
